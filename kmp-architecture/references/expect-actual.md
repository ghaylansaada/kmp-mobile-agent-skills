# Expect/Actual Patterns

## When to Use expect/actual vs Interfaces

| Criteria | expect/actual | Interface + DI |
|---|---|---|
| Constructor differs per platform | Yes | No -- interfaces have no constructors |
| Need platform types in signature | Yes | Only if in a platform source set |
| Singleton or object | Yes -- `expect object` | Possible via DI `single {}` |
| Testability in commonTest | Harder -- actual is compile-time resolved | Easier -- inject a fake |
| Multiple implementations at runtime | No -- one actual per platform | Yes -- strategy pattern |
| Third-party library wrapping | Prefer expect/actual for platform SDK types | Prefer interface for multiplatform libs |

**Rule of thumb:** Use expect/actual when common code needs to name a platform type or when constructor signatures must differ. Use interfaces for runtime polymorphism or test doubles.

## Declaration Forms

| Form | Use when |
|---|---|
| `expect class` | Need different constructor parameters per platform |
| `expect fun` | Simple factory function returning a common type |
| `expect object` | Singleton with platform-specific initialization (e.g., Room constructor) |
| `actual typealias` | Platform type matches the expect signature exactly (same supertypes, same members) |

## Preferred Alternatives to expect class

Where possible, prefer these patterns over raw `expect class`:

- **Interface + factory function:** Define the contract as an interface in commonMain, then use `expect fun createFoo(): FooInterface` to obtain platform instances. This keeps commonMain testable with fakes.
- **sealed interface for result types:** Use `sealed interface` in commonMain for closed type hierarchies. These require no expect/actual at all since they are pure common code.
- **value class for type-safe wrappers:** Use `value class` in commonMain for lightweight wrappers (IDs, tokens). These inline at compile time and avoid expect/actual overhead.

## Platform Annotation Rules

- `@OptIn(ExperimentalForeignApi::class)` -- Required for Kotlin/Native C-interop APIs (pointer ops, `memcpy`, `usePinned`, `addressOf`, `memScoped`)
- `@OptIn(BetaInteropApi::class)` -- Required for `NSData.create(bytes:length:)` with pinned Kotlin arrays
- Apply `@OptIn` at narrowest scope (function level, not file or class)
- Never add `@OptIn` to the `expect` declaration in commonMain

## Foundation Pairs

### PlatformContext

```kotlin
// commonMain -- core/platform/PlatformContext.kt
package {your.package}.core.platform

expect class PlatformContext
```

```kotlin
// androidMain -- core/platform/PlatformContext.android.kt
package {your.package}.core.platform

import android.content.Context

actual class PlatformContext(val androidContext: Context)
```

```kotlin
// iosMain -- core/platform/PlatformContext.ios.kt
package {your.package}.core.platform

actual class PlatformContext
```

iOS does not require a Context abstraction. The class is an empty marker satisfying the shared `expect` declaration.

### getPlatform()

```kotlin
// commonMain -- core/platform/Platform.kt
package {your.package}.core.platform

interface Platform {
    val deviceModel: String
    val manufacturer: String
    val osType: OsType
    val osVersionCode: Int
    val osVersionName: String

    enum class OsType { ANDROID, IOS }
}

expect fun getPlatform(): Platform
```

```kotlin
// androidMain -- core/platform/Platform.android.kt
package {your.package}.core.platform

import android.os.Build

class AndroidPlatform : Platform {
    override val deviceModel: String = Build.MODEL ?: "Android"
    override val manufacturer: String = Build.MANUFACTURER ?: "Unknown"
    override val osType: Platform.OsType = Platform.OsType.ANDROID
    override val osVersionCode: Int = Build.VERSION.SDK_INT
    override val osVersionName: String =
        Build.VERSION.RELEASE ?: Build.VERSION.SDK_INT.toString()
}

actual fun getPlatform(): Platform = AndroidPlatform()
```

```kotlin
// iosMain -- core/platform/Platform.ios.kt
package {your.package}.core.platform

import platform.UIKit.UIDevice

class IOSPlatform : Platform {
    private val device = UIDevice.currentDevice
    override val deviceModel: String = device.model
    override val manufacturer: String = "Apple"
    override val osType: Platform.OsType = Platform.OsType.IOS
    override val osVersionName: String = device.systemVersion
    override val osVersionCode: Int =
        device.systemVersion.substringBefore('.').toIntOrNull() ?: 0
}

actual fun getPlatform(): Platform = IOSPlatform()
```

### AppDatabaseConstructor (KSP-Generated)

Room KMP requires an `expect object` implementing `RoomDatabaseConstructor<AppDatabase>`. Actuals are generated by Room KSP -- no hand-written actual files exist.

```kotlin
// commonMain -- data/local/AppDatabaseConstructor.kt
package {your.package}.data.local

import androidx.room.RoomDatabaseConstructor

@Suppress("NO_ACTUAL_FOR_EXPECT")
expect object AppDatabaseConstructor : RoomDatabaseConstructor<AppDatabase> {
    override fun initialize(): AppDatabase
}
```

`@ConstructedBy(AppDatabaseConstructor::class)` on `AppDatabase` triggers KSP generation. Ensure KSP is configured for **all** targets in `build.gradle.kts`.

### NSData <-> ByteArray (iosMain helpers)

```kotlin
import kotlinx.cinterop.ExperimentalForeignApi
import kotlinx.cinterop.BetaInteropApi
import kotlinx.cinterop.addressOf
import kotlinx.cinterop.memScoped
import kotlinx.cinterop.refTo
import kotlinx.cinterop.usePinned
import platform.Foundation.NSData
import platform.Foundation.create
import platform.posix.memcpy

@OptIn(ExperimentalForeignApi::class)
internal fun NSData.toByteArray(): ByteArray =
    ByteArray(length.toInt()).also { dest ->
        memScoped { memcpy(dest.refTo(0), bytes, length) }
    }

@OptIn(ExperimentalForeignApi::class, BetaInteropApi::class)
internal fun ByteArray.toNSData(): NSData = this.usePinned { pinned ->
    NSData.create(bytes = pinned.addressOf(0), length = size.toULong())
}
```

The lambda parameter is named `dest` (not `bytes`) to avoid shadowing the NSData `bytes` property.
